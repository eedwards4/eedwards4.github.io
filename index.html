TRAVERSE

A VIDEOGAME CREATED FOR CS370 AT SSU

IT IS THE DISTANT POST APOCOLYPTIC FUTURE OF 2011. HUMANITY IS DEAD. AND YET SOMEHOW ONE MAN SURVIVES. HIS GOAL? FIGURE OUT WHAT THE HECK IS GOING ON. HE MUST TRAVERSE THE WARPED LANDSCAPE, SURVIVE ENEMIES, AND DESTROY THE ULTIMATE ENEMY: THE AGENT.

THE TEAM

  • Seth Nuzum: Progamming, Soundtrack, FX
  • Adam Erskine: Final Boss, Soundtrack
  • Richard Yocke: Enemy Design/Programming, Character Design/Programming
  • Ethan Edwards: Level design, Asset Design, Progamming

PERSONAL NOTES

This was a super fun project to do, we had a 4-week time limit to create the best game we could, and I think we delivered. While basic, Traverse is a fun little side-scroller with some of the elements seen in production titles like Hollow Knight and the Super Mario series.
For my personal part, I focused mainly on level/asset design, neither of which are my forte, so it was fun to learn a bit about those parts of a game development pipeline.

PERCEPTRON

MY CURRENT SUMMER PROJECT

A combination of a large laungage model and computer vision, Perceptron can ride shotgun as a user interacts with the real world and give them real time information on what they see.

PERSONAL NOTES

I plan to use Monocle, OpenCV, and a system of interlinked LLMs to create a companion, kind of like Apple's Siri but with the ability to see what you can see. This project is currently in it's very early stages so there isn't much more information than that for now.

CLUTTER BROWSER

SHOULD BE NAMED NO-CLUTTER BROWSER

A browser that lets the user have a functionally infinite amount of tabs open at once. Yeah this is gonna be a rough one.

PERSONAL NOTES

This is my backup summer project in the event that something stops me from creating Perceptron. Super sparse on the details but I can say that I've grown a bit annoyed with the tab limitations of modern browsers like firefox and chrome. Creating a browser like Clutter is a necessity if I want to continue my tab-hoarding tendencies.
